# Contributing to the Mill Road Winter Fair App

First of all — thank you for your interest in contributing to the project!

The Mill Road Winter Fair App is a volunteer-led community project created to support one of Cambridge's best-known community events. Contributions of all sizes are welcome, whether that's fixing bugs, improving documentation, suggesting features, or contributing code.

## Before You Start

Please read the project's README for information about:

- Local development setup
- Required software and SDKs
- Environment variables
- Running the app locally

## Reporting Bugs or Suggesting Features

If you've found a bug or have an idea for a feature improvement, please open a GitHub Issue.

When reporting bugs, please try to include:

- A clear description of the issue
- Steps to reproduce it
- Expected behaviour
- Screenshots or logs where relevant
- Device/platform information where applicable

## Development Workflow

1. Create a branch from `main`
2. Make your changes
3. Test your changes
4. Submit a detailed Pull Request

### Branch Naming

Where possible, branch names should reference the related issue number.

For example:

```txt
42-fix-map-marker-crash
57-add-accessibility-labels
```

If there is no existing issue, use a short descriptive branch name.

### Pull Requests

When submitting a Pull Request, please include:

* A summary of the change
* The issue number it relates to (if applicable)
* Details of how the change was tested, including on which platform(s)
* Any screenshots for UI-related changes
* Information about any new dependencies introduced

Please keep Pull Requests focused and reasonably scoped where possible.

### Testing

New features and bug fixes should include appropriate tests where practical.

Before submitting a Pull Request, please ensure:

* The app builds successfully
* Existing tests continue to pass
* New functionality has been tested locally
* Code Style

Formal linting and formatting rules are currently being introduced to the project.

For now, please aim to:

* Keep code readable and consistent
* Follow existing patterns in the codebase
* Write clear and descriptive commit messages
* Community Expectations

Please be respectful and constructive when interacting with other contributors. This is a community-focused volunteer project, and we want the repository to remain welcoming to contributors of all experience levels.

## Questions

If you're unsure about something before starting work, feel free to open an Issue for discussion first.

Thanks again for helping improve the Mill Road Winter Fair App!
